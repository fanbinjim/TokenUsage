检查日期：2026-07-11。

## 对比基准

- 当前 Windows 版截图：`codex-clipboard-dd83ba1f-58e0-4ebb-b78a-a0246d7452f7.png`
- 参考界面：`ref/codexU/docs/screenshot-0.2.0.png`、`screenshot-v0.3.0-usage.png`、`screenshot-v0.3.0-today.png`、`screenshot-v1.0.2-status-bar-customization.png`
- 长期视觉规则：`ref/codexU/docs/DESIGN_SYSTEM.md`

## 结论

当前版本是可运行的数据读取原型，但还不是参考程序的 Windows 视觉移植。最大差距不在颜色，而在窗口框架、玻璃材质、首屏信息层级和隐私展示边界。

| 维度 | 参考程序 | 当前 TokenUsage | 影响 | 优先级 |
| --- | --- | --- | --- | --- |
| 窗口框架 | 无系统标题栏、圆角悬浮窗、自定义顶部控制区 | 白色 Windows 原生标题栏，系统最小化/最大化/关闭按钮可见 | 首先破坏原生玻璃工具感 | P0 |
| 背景材质 | 桌面壁纸透出，窗口/分组/卡片有克制的玻璃层级 | 深蓝色不透明网页背景与深色渐变卡片 | 看起来像网页而非桌面工具 | P0 |
| 首屏结构 | 左侧双额度环；右侧今日/7日/累计卡；下方价值进度 | 大标题、两张额度占位卡、四张普通统计卡 | 信息层级与扫描路径完全不同 | P0 |
| 标题栏操作 | 图标化语言、主题、置顶、刷新、关闭 | 文字按钮“刷新/设置/切换主题” | 密度低，且不符合参考交互 | P1 |

| 诊断展示 | 用户语言、局部降级说明 | `localOnly` 与英文错误直接暴露 | 不一致且不够产品化 | P1 |
| 隐私 | 默认聚合、脱敏名称、路径尾名 | 线程标题可能直接显示用户任务/prompt 文本 | 违反参考设计的隐私边界 | P0 |

## 已确认的技术原因

### 窗口和系统控制按钮

`src-tauri/tauri.conf.json` 当前没有启用无边框透明窗口，因此 Windows 保留系统标题栏和最小化、最大化、关闭按钮。要接近参考图，需要：

1. 使用无边框窗口，隐藏系统标题栏。
2. 在 React 标题栏中提供拖拽区域。
3. 自行实现最小化、最大化/还原、关闭到托盘、置顶、刷新等操作。
4. 保证双击标题栏、窗口拖拽、键盘操作和托盘恢复均可用。

### 背景不透明

`src/styles.css` 将页面背景固定为 `#0b1020`，卡片又使用深色线性渐变；即使顶部有 `backdrop-filter`，底层仍然是不透明的，无法形成参考图中的桌面透出效果。

需要同时调整桌面窗口与 WebView 内容：

1. 窗口层启用透明并接入 Windows Mica/Acrylic 类原生背景效果。
2. WebView 根节点保持透明，避免设置不透明的全屏背景色。
3. 引入 Window、Section、Card、Control 四级语义 Surface token，以低透明度填充、细描边和克制阴影区分层级。
4. 移除当前大面积深色渐变和装饰性背景效果；透明不能以牺牲文字对比度为代价。

### 信息结构不一致

当前首屏先展示大号 `Codex` 标题和两张额度卡；参考程序的主视觉是“左侧双环额度 + 右侧三张 Token 卡 + 羊毛进度”，其后才是带 Tab 的工作台。

Windows 重构的目标结构：

```text
自定义标题栏：品牌 | Runtime | 主题/语言 | 置顶 | 刷新 | 关闭
└─ 概览玻璃分组
   ├─ 左：5h/7d 双环、剩余比例与重置时间
   └─ 右：今日 / 近 7 日 / 累计三张 Token 卡
          └─ API 等效价值进度
└─ 工作台玻璃分组
   ├─ Tab：今日任务 | 用量趋势 | 项目排行 | Skill
   └─ 当前 Tab 内容
└─ 底部：刷新时间、快捷键、非阻塞诊断
```

## Windows 适配原则

- 不逐像素仿制 macOS 红黄绿控制按钮；Windows 版采用无边框 + 自定义 Fluent 风格图标按钮。
- 隐藏系统三按钮后，必须保留明确的最小化、最大化/还原、关闭到托盘和退出入口。
- 快捷浮窗和主窗口应共享相同的玻璃 token、额度环模型和数据口径。
- Windows 通知区不能复刻 macOS 长状态栏文本；保留双环图标、tooltip、快捷浮窗和原生右键菜单。
- 暗色、浅色、高对比度和“减少动态效果”下均要提供可读的回退样式。

## 重构顺序

1. **P0：窗口壳与隐私**：无边框透明窗口、自定义标题栏、关闭到托盘、线程标题脱敏。
2. **P0：视觉 token 与概览区**：四级玻璃 Surface、双环额度、三张 Token 卡、价值进度。
3. **P1：固定工作台**：Tab 容器、任务看板、趋势/项目/Skill 空状态和滚动策略。
4. **P1：状态与设置**：统一中英文诊断、独立设置窗口、与托盘共享展示模型。
5. **P2：动效与细节**：轻量状态转场、进度动画、Windows/Mica 回退与可访问性验证。

## 验收标准

- 主窗口不出现系统标题栏或系统三按钮；所有替代操作可访问。
- 桌面背景可从窗口边缘和 Surface 层次中感知，卡片仍保持足够对比度。
- 首屏在 1000×760 下不出现整个页面的垂直滚动条；各分组固定且对齐。
- 缺失额度显示中性空轨道和 `--`，不能填充为 0。
- 默认界面不展示完整 prompt、工具参数、完整本地路径或原始日志。
- 与参考程序一致地把 Token、额度、任务和诊断分为不同层级，而不是混合成一个列表页面。
